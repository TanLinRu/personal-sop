# Testing

## TDD Workflow

1. Write failing test first
2. Write minimal code to pass
3. Refactor

## Coverage

- Minimum 80% coverage for new code
- Critical paths: 100%

## Test Types

| Type | Scope | Tools |
|------|-------|-------|
| Unit | Single component | JUnit, pytest |
| Integration | Component interaction | SpringBootTest |
| E2E | Full user flow | Playwright, Cypress |
