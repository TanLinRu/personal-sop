## 快速查阅

### 命名规约

```
【强制】
- 包名：小写字母
- 类名：大驼峰 (UpperCamelCase)
- 方法名/变量名：小驼峰 (lowerCamelCase)
- 常量：全大写下划线 (UPPER_SNAKE_CASE)
- 避免用 $ _ 开头结尾

【推荐】
- 抽象类：Abstract 或 Base 开头
- 异常类：Exception 结尾
- 测试类：Test 结尾
```

### OOP 规约

```
【强制】
- 覆写方法必须加 @Override
- 禁止用 final 类做可变参数
- Object.equals() 需先判 null
- 使用工具类替代可变参数构造

【推荐】
- 优先使用组合而非继承
- 静态方法不调用非静态成员
```

### 并发处理

```
【强制】
- 单例对象需完全无状态
- 线程资源必须通过线程池获取
- 必须使用线程安全集合
- 并发修改需加锁

【推荐】
- 使用 ConcurrentHashMap
- 考虑 ThreadLocal
```

### 控制语句

```
【强制】
- switch 必须有 default
- 循环体内避免 try-catch

【推荐】
- 避免取反逻辑
- 减少嵌套层级
- 避免 if-else 过多
```

### 异常日志

```
【强制】
- 异常必须捕获或抛出
- 事务必须回滚
- finally 块不 return

【推荐】
- 异常分类处理
- 避免 catch Exception
- 使用日志记录而非打印
```

### 日志规约

```
【强制】
- 应用日志必须分类输出
- 禁止 println

【推荐】
- 使用日志门面 (SLF4J)
- 日志级别正确使用
- 避免日志性能损耗
```

### 测试规约

```
【推荐】
- 单元测试覆盖率 > 80%
- 测试用例完整：Given-When-Then
- Mock 正确使用
- 测试代码与生产代码同等重要
```

### 安全规约

```
【强制】
- 禁止在 URL 参数明文传密码
- 敏感数据必须脱敏
- 禁止 SQL 拼接
- 权限必须校验
```

### 工程结构

```
【强制】
- 分层清晰：controller/service/repository
- 禁止循环依赖
- 二方库依赖需固定版本
- 配置与代码分离
```

### MySQL 规约

```
【强制】
- 表名/字段名小写下划线
- 主键 ID 用 bigint unsigned
- 禁用外键和物理删除
- 索引命名 idx_/uniq_

【推荐】
- 适当冗余字段
- 分页优化
- 避免 select *
```
