# BACKEND.md - 后端架构规范 (v2.1)

## 流程 (v2.1)

```
需求确认 → 需求调研 → PRD → 架构设计 → 架构审核 → 依赖查询 → 后端实现 → 验证 → 知识更新
```

## 步骤说明

| Step | 说明 | 关键输出 |
|------|------|----------|
| 1 | 需求确认 | 功能需求文档 |
| 2 | 需求调研 | 3次调研 |
| 3 | 生成 PRD | PRD 文档 |
| 4 | 架构设计 | API/数据模型 |
| 5 | 架构审核 | 审核通过 |
| 6 | 依赖查询 | 避免冲突 |
| 7 | 后端实现 | 代码 |
| 8 | 验证 | 编译测试 |
| 9 | 知识更新 | 知识库 |

## 多 Agent 配置

| Agent | 用途 | 调用次数 |
|-------|------|---------|
| sop-library-research | 调研 | 3次 |
| dr-jskill | 后端生成 | 1次 |
| java-reviewer | 代码审查 | 1次 |
