# SOP Brainstorming — 结构化头脑风暴

## 概述

引导用户进行**结构化头脑风暴**，通过发散思考 → 收敛筛选 双阶段，产出可落地的想法列表。支持多种脑暴方法，用户可选或 Agent 自动推荐。

**核心理念**：
- **先发散，后收敛** — 先不设限地产生想法，再分类筛选
- **方法驱动** — 不是漫无边际聊天，而是用结构化方法提高产出质量
- **结果可落地** — 脑暴结果自动整理为结构化文档

## 工作流程

```
Step 1: 确定议题
   ↓ 明确脑暴主题、目标、约束
Step 2: 选择方法
   ↓ Agent 推荐或用户选择脑暴方法
Step 3: 发散阶段
   ↓ 围绕方法引导产生大量想法（不评判、不筛选）
Step 4: 收敛阶段
   ↓ 分类 → 合并 → 优先级排序 → 可行性评估
Step 5: 输出结果
   ↓ 结构化的脑暴报告
```

## 步骤详情

### Step 1: 确定议题 ⭐ [CONFIRM_REQUIRED]
> **auto_default**: SKIP — 需要人定主题

**目标**：明确脑暴的主题、目标和约束边界

Agent 引导用户回答：
- **主题**：我们要围绕什么话题进行脑暴？（如：新功能名称、产品方向、问题解决方案）
- **目标产出**：期望得到什么？（如：10 个候选名称、3 个可行方案、优先级列表）
- **约束条件**：有什么限制？（时间、预算、技术、资源）

> **平台交互**：
> - OpenCode：弹窗一次性收集议题、目标、约束
> - Claude：自然语言引导用户描述

```
Agent: 好的，我们开始一次结构化脑暴！请先告诉我：

1️⃣ 脑暴主题是什么？（例如：为物流配送 App 起名）
2️⃣ 目标产出是什么？（例如：5 个候选名称）
3️⃣ 有什么约束条件？（例如：不超过 4 个字、中文）
```

### Step 2: 选择方法 [AUTO]

**目标**：选择适合议题的脑暴方法

Agent 根据议题自动推荐方法，也允许用户选择：

| 方法 | 适合场景 | 产出类型 |
|------|----------|----------|
| **自由联想** | 通用，无特定方向 | 大量发散想法 |
| **SCAMPER** | 已有产品/方案优化 | 改进方向列表 |
| **六顶思考帽** | 复杂问题全面分析 | 多维度分析报告 |
| **头脑书写** | 团队协作（远程友好） | 独立思考 + 组合想法 |
| **思维导图** | 主题分解，结构探索 | 层级结构图 |
| **反向脑暴** | 找问题/风险 | 潜在问题列表 |

> **平台交互**：Agent 列出方法及说明，弹窗/自然语言选择

### Step 3: 发散阶段 [AUTO]

**目标**：围绕议题产生大量想法，不评判、不筛选

Agent 根据选择的方法引导用户逐轮发想：

**通用发散规则**：
- **不评判**：所有想法都是好想法，延迟评判
- **追求量**：先数量，后质量
- **自由联想**：鼓励大胆、不寻常的想法
- **组合改进**：可以在他人的想法上组合、改进

**方法引导**：
```
┌─ 自由联想 ──────────────────────────┐
│ 围绕"XX"你能想到什么？不用思考好坏，  │
│ 先说5个关键词。                       │
│ → [用户回复]                         │
│ 很好！再试试关联这些词还能想到什么？   │
└─────────────────────────────────────┘

┌─ SCAMPER ───────────────────────────┐
│ 针对"XX"，试试 Substitute（替代）：  │
│ 有什么可以用其他材料/流程替代的？      │
│ → [用户回复]                         │
│ 再试试 Combine（组合）：              │
│ 可以和什么功能/模块组合？             │
└─────────────────────────────────────┘
```

**每轮引导后 Agent 主动记录想法**：
```markdown
## 当前想法清单

| # | 想法 | 来源 |
|---|------|------|
| 1 | ... | 自由联想 |
| 2 | ... | SCAMPER - Substitute |
```

### Step 4: 收敛阶段 [AUTO]

**目标**：对发散阶段的想法进行分类、筛选、优先级排序

**收敛步骤**：

1. **分类** — 按主题/维度分组
2. **合并** — 去重，合并相似想法
3. **评估** — 对每组想法评估：
   - 🎯 **可行性**：高 / 中 / 低
   - 💡 **创新性**：高 / 中 / 低
   - ⚡ **影响度**：高 / 中 / 低
4. **排序** — 根据综合得分排序

```
┌─ 收敛结果示例 ──────────────────────┐
│                                      │
│ 分组 A：UI 改进类                    │
│   ├ 想法 1：暗黑模式               │
│   │  可行性: 高  创新性: 低  推荐 ✅ │
│   └ 想法 2：手势滑动操作            │
│      可行性: 中  创新性: 高  推荐 ✅ │
│                                      │
│ 分组 B：功能扩展类                   │
│   └ 想法 3：社交分享                 │
│      可行性: 低  创新性: 中  暂缓    │
└─────────────────────────────────────┘
```

### Step 5: 输出结果 [AUTO] [VERIFY]

**目标**：生成结构化脑暴报告

```markdown
# 脑暴报告：${主题}

## 基本信息
- **日期**: ${date}

- **目标**: ${goal}

## 发散阶段

## 收敛阶段

|------|--------|--------|--------|
| ...  | 高     | 高     | 高     |

### 可考虑
| 想法 | 备注 |
|------|------|
| ...  | ...  |

### 暂缓/淘汰
| 想法 | 原因 |
|------|------|
| ...  | ...  |

## 下一步行动
- [ ] ...
```
