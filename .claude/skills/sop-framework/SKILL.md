# SOP 自动化执行框架

## 概述

本框架为 SOP 提供统一的自动化执行能力，实现：
- **核心步骤** 需要用户确认
- **非核心步骤** 自动执行
- **模式切换** 支持手动/自动模式
- **断点恢复** 中断后从上次步骤继续
- **执行可视化** 实时显示进度

## 断点恢复 (Resume Detection)

在执行任何 SOP 前，先检查是否有未完成的任务：

```bash
npx ts-node --transpile-only .claude/scripts/sop-resume-check.ts <sop-name>
```

**如果返回 `has_resume: true`**：
1. 向用户展示："发现进行中的 <sop> 任务，停在步骤 <N>: <step_name>。是否从这里继续？(Y/n)"
2. 用户确认 → 跳到 `current_step`，使用已保存的 `answers`
3. 用户拒绝 → 运行 `npx ts-node --transpile-only .claude/scripts/sop-state-clean.ts <sop-name>` 清理旧状态，重新开始

**如果返回 `has_resume: false`**：正常从步骤 1 开始。

## 执行可视化 (Progress Display)

每步开始时，渲染进度表：

```markdown
### SOP 进度: <sop-name>

| # | 步骤 | 状态 |
|---|------|------|
| 1 | 需求确认 | completed |
| 2 | 环境配置 | completed |
| 3 | 代码生成 | **running** |
| 4 | 验证测试 | pending |

Step 3/6: 代码生成 [==============>................] 50%
```

每步完成后，渲染完成标记：

```markdown
### Step 3: 代码生成 [COMPLETED]

耗时: 2m 15s
产出: `.sop/output/scaffold-delivery-staff.md`

进入 Step 4: 验证测试...
```

SOP 全部完成后，自动生成执行报告到 `.sop/output/{sop}-{date}-report.md`。

## 框架设计

### 步骤类型定义

| 步骤类型 | 标识 | 说明 | 自动化行为 |
|----------|------|------|------------|
| **核心步骤** | `⭐` 或 `[CONFIRM_REQUIRED]` | 关键决策点，需要用户确认 | 暂停等待确认 |
| **自动执行** | `[AUTO]` | 例行操作，可自动执行 | 直接执行 |
| **信息收集** | `[INFO]` | 收集信息，不阻塞流程 | 自动执行 |
| **验证步骤** | `[VERIFY]` | 需要验证结果 | 暂停等待验证 |

### 执行模式配置

执行模式由两个正交维度组成：

```yaml
# SKILL.md frontmatter 中配置
execution:
  mode: sequential  # 任务执行模式: sequential | parallel | hybrid
  timeout: 300000
  checkpoint_dir: .sop/state
  state_file: .sop/state/{sop}-{id}.json
```

### 双维度模式

| 维度 | 配置项 | 可选值 | 说明 |
|------|--------|--------|------|
| **交互模式** | 由步骤标记决定 | `confirm` / `auto` | `[CONFIRM_REQUIRED]` 暂停确认，`[AUTO]` 自动执行 |
| **执行模式** | `execution.mode` | `sequential` / `parallel` / `hybrid` | 任务间的执行顺序 |

### 交互模式

由步骤标记控制，无需单独配置：

| 步骤标记 | 行为 | 适用场景 |
|----------|------|----------|
| `⭐ [CONFIRM_REQUIRED]` | 暂停等待用户确认 | 关键决策点 |
| `[AUTO]` | 自动执行 | 例行操作 |
| `[INFO]` | 信息展示，不阻塞 | 收集信息 |
| `[VERIFY]` | 展示结果，等待验证 | 检查点 |

### 执行模式

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| `sequential` (默认) | 步骤按顺序执行 | 大多数 SOP |
| `parallel` | 独立步骤并行执行 | 多 Agent 协作 |
| `hybrid` | 部分并行，部分串行 | 故障响应等 |

### 自动模式切换指令

用户可通过以下方式切换模式：

```bash
# 切换到自动模式（关闭确认提示）
/sop xxx --auto

# 或在 SOP 执行过程中
> 后续步骤全部自动执行

# 切换回确认模式
> 后续步骤需要确认
```

## 步骤标注规范

在每个步骤的 **执行内容** 部分使用标注：

```markdown
### 步骤一：需求确认 ⭐ [CONFIRM_REQUIRED]

**目标**：确认业务需求

**执行内容**：
1. 确认功能需求 [AUTO]
2. 确认技术约束 [CONFIRM_REQUIRED]
3. 确认时间计划 [CONFIRM_REQUIRED]

**用户确认**：
- [ ] 需求理解正确
- [ ] 技术方案认可
- [ ] 时间计划可行
```

### 步骤状态自动流转

```markdown
---
sop: xxx
step: 1_confirm
status: pending  # pending → confirmed → completed
---
```

| 状态 | 说明 | 自动化处理 |
|------|------|------------|
| `pending` | 待执行 | 显示步骤，等待用户 |
| `in_progress` | 执行中 | 自动流转 |
| `confirmed` | 已确认 | 核心步骤用户确认后更新 |
| `completed` | 已完成 | 自动流转到下一步 |

## 执行引擎指令

### 自动执行触发

当用户选择自动模式时，SOP 执行引擎自动：

1. **跳过确认** - 直接执行非核心步骤
2. **自动流转** - 完成一步后自动进入下一步
3. **使用默认值** - 对不确定的选项使用推荐值

### 指令示例

```markdown
### 步骤二：环境检查 [AUTO]

**目标**：自动检查开发环境

**执行内容**：
1. 检查 JDK 版本 [AUTO]
2. 检查 Maven 版本 [AUTO]
3. 检查 Node.js 版本 [AUTO]

**执行指令**：
```bash
# 自动执行
java -version
mvn -version
node -version
```

**输出**：
```markdown
---
sop: scaffold
step: 2_env_check
status: completed
---

## 环境检查结果

| 组件 | 要求 | 实际 | 状态 |
|------|------|------|------|
| JDK | 17+ | 21 | ✅ |
| Maven | 3.8+ | 3.9 | ✅ |
| Node.js | 18+ | 20 | ✅ |

→ 自动进入下一步：项目生成
```
```

## 自动执行示例

### 手动模式（默认）

```
用户: /sop scaffold
SOP: 请确认以下信息：
- 项目名称: [输入]
- 技术栈: [选择]
- 输出目录: [输入]
用户: my-app, Spring Boot + Vue 3, ./
SOP: 步骤一已完成，请确认技术方案...
用户: 确认
SOP: [自动执行后续步骤...]
```

```
用户: /sop scaffold --auto

- 步骤一：需求分析 [自动使用默认值]
- 步骤二：后端生成 [自动生成]
- 步骤三：前端生成 [自动生成]
- 步骤四：API 文档 [自动配置]
- 步骤五：配置联调 [自动配置]
- 步骤六：验证运行 [自动验证]
完成！

## 实施建议

1. **分阶段实施**
   - 第一阶段：定义所有 SOP 的核心确认点
   - 第二阶段：实现自动执行逻辑
   - 第三阶段：添加模式切换功能

2. **优先级顺序**
   - 高自动化：sop-scaffold, sop-testing, sop-deployment
   - 中自动化：sop-code-review, sop-bug-fix, sop-knowledge
   - 低自动化：sop-product-analysis, sop-incident-response

3. **持续优化**
   - 收集自动模式的反馈
   - 调整核心确认点
   - 增加自动执行范围
