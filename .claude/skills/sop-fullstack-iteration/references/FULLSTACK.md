# FULLSTACK.md - 全栈架构规范 (v2.1)

## 流程 (v2.1)

```
需求确认 → 需求/技术调研 → PRD → 架构设计 → 架构审核 → 依赖查询 → 并行生成 → 联调测试 → 知识更新
```

## 步骤说明

| Step | 说明 | 关键输出 |
|------|------|----------|
| 1 | 需求确认 | 完整需求文档 |
| 2 | 需求调研 | 5次调研 |
| 3 | 生成 PRD | PRD 文档 |
| 4 | 架构设计 | 前后端架构 |
| 5 | 架构审核 | 审核通过 |
| 6 | 依赖查询 | 避免冲突 |
| 7 | 并行生成 | 前后端代码 |
| 8 | 联调测试 | 验证通过 |
| 9 | 知识更新 | 知识库 |
