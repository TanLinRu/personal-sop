# FRONTEND.md - 前端架构规范 (v2.1)

## 流程 (v2.1)

```
需求确认 → UI/UX调研 → 组件设计 → 设计审核 → 依赖查询 → 前端实现 → 验证 → 知识更新
```

## 步骤说明

| Step | 说明 | 关键输出 |
|------|------|----------|
| 1 | 需求确认 | 页面需求文档 |
| 2 | UI/UX调研 | 2次调研 |
| 3 | 组件设计 | 组件树 |
| 4 | 设计审核 | 审核通过 |
| 5 | 依赖查询 | 避免冲突 |
| 6 | 前端实现 | 代码 |
| 7 | 验证 | 构建测试 |
| 8 | 知识更新 | 知识库 |

## 多 Agent 配置

| Agent | 用途 | 调用次数 |
|-------|------|---------|
| sop-library-research | 调研 | 2次 |
| frontend-design | 前端生成 | 1次 |
| code-reviewer | 代码审查 | 1次 |
