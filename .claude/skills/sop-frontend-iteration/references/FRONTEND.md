# FRONTEND.md - 前端架构规范 (v2.0)

## 流程图

```
需求确认 → 依赖查询 → UI/UX调研(2) → 组件设计 → 设计审核 → 前端实现 → 验证 → 知识更新
```

## 多 Agent 配置

| Agent | 用途 | 调用次数 |
|-------|------|---------|
| sop-library-research | 调研 | 2次 |
| frontend-design | 前端生成 | 1次 |
| code-reviewer | 代码审查 | 1次 |

## 调研内容

| 领域 | 内容 |
|------|------|
| UI设计 | 设计系统、布局、主题 |
| 组件库 | 组件选择、状态管理 |

## 设计审核检查点

### P0 (阻塞级)
- 组件结构合理可复用
- 路由设计 RESTful 风格
- 包结构按页面/组件划分

### P1 (重要级)
- 状态管理集中
- TypeScript 类型安全
- 性能优化

### P2 (优化级)
- 可访问性 WCAG 2.1
- 响应式适配
