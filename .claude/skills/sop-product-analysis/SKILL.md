# SOP Product Analysis - 产品业务分析流程

## 概述

本 SOP 提供标准化的产品业务分析流程，确保在项目立项之初进行完整的业务梳理、竞品分析和架构设计。避免技术先行而忽视业务真实需求。

## 工作模式

本 SOP 支持两种工作模式：

### 模式一：完整流程（推荐）
适用于全新项目立项：业务理解 → 竞品分析 → 用户旅程 → 模块划分 → 架构设计 → 输出文档

### 模式二：PRD 快速生成
适用于快速生成标准 PRD 文档

> **提示**：使用 `/prp-prd` 命令可直接调用外部 PRD 生成工作流（来自 affaan-m/everything-claude-code）

## 使用场景

- 全新项目立项
- 现有系统重构
- 新功能规划
- 竞品对标分析
- 架构升级评估

## 为什么需要业务分析

**当前问题**：
- 项目立项时缺少业务梳理，架构设计「拍脑袋」
- 技术选型与服务业务目标脱节
- 模块边界不清晰，后期难以扩展

**解决价值**：
- 通过竞品分析了解行业标准功能和最佳实践
- 基于业务用例驱动模块划分
- 确保技术架构服务于业务目标

### 定量竞品评分（加权评分矩阵）

> 对主要竞品进行定量评分，识别差异化机会。

| 评估维度 | 权重 | 竞品A | 竞品B | 我们 |
|----------|------|-------|-------|------|
| 功能完整性 | 25% | 4 | 3 | 2 |
| 用户体验 | 20% | 3 | 4 | 3 |
| 性能/可靠性 | 15% | 3 | 3 | 4 |
| 定价竞争力 | 15% | 2 | 4 | 3 |
| 技术先进性 | 10% | 3 | 3 | 4 |
| 生态/集成 | 10% | 4 | 2 | 2 |
| 文档/社区 | 5% | 3 | 3 | 3 |
| **加权总分** | 100% | **3.25** | **3.20** | **2.95** |

**评分标准**：1=很差, 2=差, 3=一般, 4=好, 5=很好

**雷达图**（Mermaid）：
```mermaid
%%{init: {'theme': 'default'}}%%
radar
  title 竞品对比
  axis 功能完整性, 用户体验, 性能, 定价, 技术, 生态, 文档
  curve 竞品A: [4,3,3,2,3,4,3]
  curve 竞品B: [3,4,3,4,3,2,3]
  curve 我们: [2,3,4,3,4,2,3]
```

**差异化策略**：
- **领先维度**：{我们得分最高的维度} → 强化优势
- **追赶维度**：{与竞品差距最大的维度} → 重点投入
- **差异化维度**：{竞品都低分但我们有机会的维度} → 蓝海机会
```
