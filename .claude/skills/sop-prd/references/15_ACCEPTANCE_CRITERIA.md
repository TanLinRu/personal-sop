# 验收标准指南 (Given/When/Then)

## 格式说明

```
Given {前置条件}
When {用户操作}
Then {预期结果}
```

- **Given**：描述系统初始状态或前置条件
- **When**：描述用户执行的操作
- **Then**：描述系统应有的响应或结果

---

## 示例

### 示例 1：用户注册

| AC# | 验收标准 |
|-----|----------|
| AC-1 | Given 用户未注册, When 填写有效邮箱和密码并提交, Then 账号创建成功并收到验证邮件 |
| AC-2 | Given 用户未注册, When 填写已注册邮箱并提交, Then 提示"邮箱已被注册" |
| AC-3 | Given 用户未注册, When 填写弱密码(<8字符)并提交, Then 提示密码不符合要求 |
| AC-4 | Given 用户未注册, When 不填写必填字段并提交, Then 提示"请填写必填字段" |

### 示例 2：订单支付

| AC# | 验收标准 |
|-----|----------|
| AC-1 | Given 用户有未支付订单, When 选择支付方式并确认支付, Then 订单状态变为"已支付" |
| AC-2 | Given 用户有未支付订单, When 支付超时(30分钟), Then 订单自动取消 |
| AC-3 | Given 用户有未支付订单, When 余额不足, Then 提示"余额不足"并保留订单 |
| AC-4 | Given 支付过程中网络中断, When 用户重新进入, Then 显示支付状态并支持重试 |

### 示例 3：数据导出

| AC# | 验收标准 |
|-----|----------|
| AC-1 | Given 用户有数据权限, When 点击"导出"并选择格式, Then 生成并下载对应格式文件 |
| AC-2 | Given 数据量超过 10000 条, When 点击"导出", Then 异步处理并通知下载 |
| AC-3 | Given 用户无数据权限, When 点击"导出", Then 提示"无导出权限" |

---

## 编写原则

### 1. 具体可测

```
❌ 模糊：系统应该快速响应
✅ 具体：When 用户点击搜索, Then 结果在 500ms 内返回
```

### 2. 独立完整

```
❌ 不完整：Then 返回错误
✅ 完整：Then 返回 HTTP 400，错误消息为"邮箱格式不正确"
```

### 3. 覆盖正向和负向

```
正向：Given 有效输入, When 操作, Then 成功
负向：Given 无效输入, When 操作, Then 错误提示
边界：Given 边界值, When 操作, Then 正确处理
```

### 4. 避免实现细节

```
❌ 实现细节：Then 数据库插入一条记录
✅ 行为描述：Then 用户列表中显示新用户
```

## 与测试用例的映射

每个验收标准 (AC) 应至少对应一个测试用例 (TC)：

| 验收标准 | 测试用例 | 测试类型 |
|----------|----------|----------|
| AC-001-1 | TC-001 | 正向测试 |
| AC-001-2 | TC-002 | 负向测试 |
| AC-001-3 | TC-003 | 边界测试 |

> 运行 `/sop test-design` 时，会自动从 AC 生成 TC 并建立追溯矩阵。
