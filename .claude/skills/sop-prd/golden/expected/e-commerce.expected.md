## 0. 执行摘要

### 问题陈述
新创 B2C 电商团队（年 GMV 目标 5000 万）需要一份完整 PRD 给投资方汇报，覆盖市场、产品、技术、运营全维度。

### Proposed Solution
基于 Vue 3 + Spring Boot 的全栈电商后台，含买家端 H5、卖家端 PC、运营端，3 个月 MVP 上线。

### 关键假设
We believe **垂直领域选品 + 强运营内容** will **达到 5% 转化率**.
We'll know we're right when **首月 DAU > 5000 + GMV > 100 万**.

### 成功指标
| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 转化率 | ≥ 5% | GA |
| 首月 DAU | ≥ 5000 | 后台日报 |
| 客单价 | ≥ 150 元 | 订单系统 |

## 1. 业务背景

### 1.1 行业背景
2025 年中国电商市场规模 14.5 万亿元，增速放缓至 8%，但**垂直内容电商**赛道增速 35%（引用 *艾瑞咨询 2025 H2*）。

### 1.2 行业挑战
- 流量成本上升（CPC 从 2 元涨到 5 元）
- 同质化严重，差异化困难
- 平台抽佣压缩利润空间

### 1.3 产品目标
- 垂直选品（聚焦 3-C / 母婴 / 美妆）
- 内容驱动转化（每商品配视频）
- 私域运营（微信生态闭环）

### 1.4 成功指标
| 指标 | 目标值 | 行业基准 |
|------|--------|----------|
| 转化率 | 5% | 3% |
| 复购率 | 35% | 22% |
| GMV | 5000 万/年 | - |

## 2. 产品概述

### 2.1 产品定位
为**新创垂直电商**提供**内容驱动 + 私域运营**的全栈电商系统。

### 2.2 目标用户
| 角色 | 描述 | 场景 |
|------|------|------|
| 买家 | 18-35 都市女性 | H5 浏览/下单 |
| 卖家 | 中小供应商 | PC 上架/发货 |
| 运营 | 内容运营 | 选品/活动 |

### 2.3 产品范围
- **包含**：商品/订单/营销/内容/会员 5 大模块
- **不包含**：物流自营、跨境结算、社交直播

### 2.4 竞品分析
| 竞品 | 优势 | 劣势 | 差异化 |
|------|------|------|--------|
| 有赞 | 成熟 SaaS | 抽佣高 | 自营 + 低费率 |
| 抖音小店 | 流量大 | 平台依赖 | 私域沉淀 |

## 3. 市场研究

### 3.1 竞品分析
聚焦内容电商：有赞、小红书商城、抖音小店三家做了深度对比，差异化在**私域闭环**。

### 3.2 行业最佳实践
- 商品详情页视频化
- 客服 AI 24h 响应
- 会员等级 + 积分

### 3.3 技术方案参考
| 方案 | 特点 | 适用性 |
|------|------|--------|
| 微服务 | 灵活，复杂 | 暂不需要 |
| 单体 + 模块化 | 快速上线 | ✓ 选择 |

## 4. 产品设计

### 4.1 信息架构
首页 → 频道页 → 商品详情 → 购物车 → 结算 → 订单。会员中心、内容流为辅。

### 4.2 核心页面设计
| 页面 | 功能 | 关键组件 |
|------|------|----------|
| 首页 H5 | 内容流 | 视频卡片、商品列表 |
| 商品详情 | 转化 | 视频、规格、评价、客服 |
| 卖家后台 | 上架 | 商品表单、库存 |

### 4.3 交互流程
关键流程：浏览 → 加购 → 下单 → 支付 → 物流。每环节降流失。

### 4.4 设计规范
Naive UI + 自定义 Token，主色 #FF6B35（暖橙），适合食品/美妆。

## 7. 技术方案

### 7.1 技术栈
Vue 3 + Naive UI（前端）, Spring Boot 3 + MyBatis-Plus（后端）, MySQL + Redis（数据）, Docker（部署）。

### 7.2 关键数据模型
| 实体 | 字段 | 关系 |
|------|------|------|
| Product | id, sku, price, stock | 1-N Sku |
| Order | id, user_id, total | 1-N OrderItem |

### 7.3 接口设计
RESTful，OpenAPI 3，详见 `/sop api-design` 产出。

### 7.4 集成方案
微信支付、阿里云 OSS、SMS 三方。
