# 游戏风控系统 - 产品业务分析报告

**项目**：游戏风控系统 (Game Risk Control System)
**分析日期**：2026-04-18
**版本**：v1.0

## 一、业务理解

### 1.1 业务背景

| 项目 | 内容 |
|------|------|
| **业务场景** | 游戏安全风控 |
| **目标用户** | 游戏厂商、运营平台 |
| **核心价值** | 识别和预防游戏内欺诈行为，保障游戏生态健康 |
| **行业定位** | 游戏安全领域的基础设施 |

### 1.2 解决的问题

| 痛点 | 影响 | 解决方案 |
|------|------|----------|
| 批量注册/虚拟号 | 垃圾账号泛滥，资源被薅 | 注册风控：虚拟号检测、人脸核身 |
| 异地登录/暴力破解 | 账号盗取、资产损失 | 登录风控：异地检测、暴力破解防护 |
| 充值欺诈/洗钱 | 资金损失、支付通道风险 | 支付风控：欺诈检测、洗钱识别 |
| 外挂/脚本/刷资源 | 游戏公平性破坏、收入流失 | 反作弊：行为分析、外挂检测 |
| 辱骂/广告/违规内容 | 用户体验下降、合规风险 | 内容风控：文本过滤、内容审核 |

### 1.3 核心业务指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| TPR（召回率） | >95% | 欺诈检出率 |
| FPR（误杀率） | <1% | 正常用户误判率 |
| 检测延迟 | <50ms | P99延迟 |
| 系统可用性 | 99.99% | 年度停机<53分钟 |
| 准确率（反作弊） | >99% | 外挂检测准确率 |
| 拦截率（内容） | >99.8% | 违规内容拦截率 |

### 1.4 约束条件

- **时间约束**：MVP 3个月完成
- **成本约束**：自建方案为主，商业方案为辅
- **技术约束**：Spring Boot + Vue 3 技术栈
- **合规要求**：GDPR/个人信息保护法合规、数据脱敏

## 二、竞品分析

### 2.1 商业竞品

| 竞品 | 核心功能 | 目标用户 | 定价模式 | 优势 | 不足 |
|------|----------|----------|----------|------|------|
| **网易易盾** | 全链路风控（注册/登录/支付/内容） | 游戏、社交、电商 | 按量/包年 | 覆盖全面、接入简单 | 成本较高 |
| **腾讯ACE** | 游戏专属风控（反外挂/反脚本） | 游戏厂商 | 按量 | 延迟低、针对性强 | 绑定腾讯生态 |
| **数美科技** | 内容风控、行为风控 | 互联网企业 | 按量 | 内容审核领先 | 游戏场景深耕不足 |
| **顶象** | 业务安全风控 | 金融、电商、游戏 | 包年 | 实时防护能力强 | 定价较高 |

### 2.2 开源方案

| 方案 | 技术栈 | 成熟度 | 社区活跃度 | 适用场景 |
|------|--------|--------|------------|----------|
| **Flink CEP** | Java | 高 | 活跃 | 实时复杂事件处理 |
| **Drools** | Java | 高 | 一般 | 企业级规则引擎 |
| **LiteFlow** | Java | 中 | 活跃 | 轻量级规则编排 |
| **Prometheus** | Go | 高 | 活跃 | 监控指标采集 |
| **ClickHouse** | C++ | 高 | 活跃 | 日志/行为数据分析 |

### 2.3 差异化定位

**市场机会**：
- 商业方案：功能全但成本高，适合大型厂商
- 开源方案：成本低但需要技术投入，适合中小厂商

**我们定位**：
- ✅ 轻量级：专注游戏场景，开箱即用
- ✅ 可定制：规则引擎可配置，业务逻辑透明
- ✅ 低成本：开源+自建混合架构
- ✅ 易集成：标准化API，快速接入

### 3.3 详细业务流程

#### 3.3.1 注册风控流程

```
玩家注册 ──► 基础信息采集 ──► 规则引擎检测 ──► 风险评分
                │                  │                   │
                ▼                  ▼                   ▼
            手机号验证        虚拟号库匹配         分数>80: 放行
            IP地址记录        批量注册检测         分数60-80: 验证码
            设备指纹          风险名单查询         分数<60: 拦截
                                │
                                ▼
                            高风险 ──► 人脸核身/短信验证
```

#### 3.3.2 登录风控流程

```
玩家登录 ──► 信息采集 ──► 实时风控 ──► 风险评估
              │              │               │
              ▼              ▼               ▼
          账号密码       暴力破解检测      正常: 放行
          登录IP         异地登录检测      异常: 验证码
          设备信息       撞库攻击检测      高风险: 锁定
          登录时间       黑名单匹配
                            │
                            ▼
                      需要验证 ──► 短信/邮件验证码
```

#### 3.3.3 支付风控流程

```
玩家支付 ──► 交易信息 ──► 规则检测 ──► ML模型 ──► 决策
              │              │            │          │
              ▼              ▼            ▼          ▼
          金额            异常金额检测   欺诈模型    通过
          商品            频繁交易检测   洗钱模型    人工审核
          支付方式        卡BIN检测      风险评分    拦截
          收货地址        风险商户检测
```

#### 3.3.4 反作弊流程

```
游戏行为 ──► 数据采集 ──► 特征工程 ──► 模型检测 ──► 处置
              │              │            │          │
              ▼              ▼            ▼          ▼
          操作日志       行为特征       外挂检测    警告
          游戏事件       序列特征       脚本检测    封号
          鼠标/键盘      统计特征       加速检测    踢出游戏
          游戏坐标       时序特征       刷资源检测
                                │
                                ▼
                           实时检测 ──► 异步分析
```

#### 3.3.5 内容风控流程

```
用户发言 ──► 内容采集 ──► 文本检测 ──► 语义分析 ──► 处置
              │              │            │          │
              ▼              ▼            ▼          ▼
          文本内容       敏感词匹配     语义分析    通过
          图片/视频      违规图案检测   上下文理解  人工审核
          语音           广告识别       意图识别    拦截
                                │
                                ▼
                           违规 ──► 记录并警告用户
```
