# 游戏风控系统技术调研报告

## 调研目标

### 调研对象
- **项目名称**: 游戏风控系统 (Game Risk Control System)
- **调研目的**: 为在线游戏（MMO/休闲/棋牌类）提供注册、登录、支付、行为反作弊全链路风控解决方案
- **替代方案**: 自建 vs 采购商业方案

### 业务场景覆盖

| 场景 | 功能描述 | 核心指标 |
|------|----------|----------|
| 注册风控 | 批量注册、虚拟号检测、人脸核身 | TPR>95%, FPR<1% |
| 登录风控 | 异地登录、暴力破解、撞库攻击 | 延迟<50ms |
| 支付风控 | 充值欺诈、洗钱、卡BIN检测 | 召回率>95% |
| 行为反作弊 | 外挂、脚本、刷资源、加速器 | 检测准确率>99% |
| 内容风控 | 辱骂、广告、敏感词过滤 | 拦截率>99.8% |

---

## 调研结论

### 决策建议
- [x] 推荐使用混合架构（自建规则引擎 + 商业ML模型）

### 技术选型矩阵

| 层级 | 技术选型 | 版本 | 决策理由 |
|------|----------|------|----------|
| 后端框架 | Spring Boot | 3.x | 成熟稳定，生态丰富 |
| 规则引擎 | Drools | 10.1.0 | Apache开源，Rete算法，支持动态规则 |
| ML模型 | LightGBM | 4.x | 训练快速，AUC 0.87+，支持不平衡数据 |
| 实时流处理 | Flink | 1.18+ | 支持CEP复杂事件处理 |
| 数据库 | MySQL 8.0 + ClickHouse | 22.x | OLTP + OLAP分离 |
| 缓存 | Redis | 7.x | 实时特征存储 |
| 前端 | Vue 3 + Element Plus | 3.x / 2.x | 轻量级UI组件 |

### 优缺点分析

**优点**:
1. Spring Boot + Drools 组合成熟度高，文档丰富
2. LightGBM 在不平衡数据（欺诈<1%）场景表现优异
3. Flink CEP 支持实时复杂事件检测
4. 开源方案可定制，成本可控

**缺点**:
1. 自建需要较强算法团队
2. 实时性要求高，需要性能优化
3. 规则和模型需要持续迭代维护
